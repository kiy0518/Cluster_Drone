# Cursor Git 확장 기능 사용 가이드

Cursor(VS Code)의 내장 Git 기능을 사용하면 명령어 없이 쉽게 GitHub에 업로드할 수 있습니다.

## 🎯 Cursor에서 Git 사용하기

### 1. 소스 제어 패널 열기

**방법 1: 아이콘 클릭**
- 왼쪽 사이드바에서 **소스 제어 아이콘** (분기 모양) 클릭
- 또는 `Ctrl+Shift+G` (Linux/Windows) / `Cmd+Shift+G` (Mac)

**방법 2: 명령 팔레트**
- `Ctrl+Shift+P` (Linux/Windows) / `Cmd+Shift+P` (Mac)
- "Git: Show Source Control" 입력

### 2. 변경사항 확인

소스 제어 패널에서:
- **변경됨 (Changes)**: 수정된 파일 목록
- **스테이징됨 (Staged Changes)**: 커밋할 파일 목록
- 각 파일 옆의 **+** 버튼: 파일을 스테이징 영역에 추가
- **모두 스테이징** 버튼: 모든 변경사항 추가

### 3. 커밋 생성

1. **커밋 메시지 입력창**에 메시지 입력
   - 예: "RTK GPS 정밀 호버링 기능 추가"
   - 예: "문서: 설치 가이드 업데이트"

2. **✓ 커밋** 버튼 클릭
   - 또는 `Ctrl+Enter` (Linux/Windows) / `Cmd+Enter` (Mac)

### 4. GitHub에 푸시

**방법 1: 동기화 버튼 사용**
- 상단의 **동기화** 버튼 (위아래 화살표) 클릭
- 또는 `Ctrl+Shift+P` → "Git: Sync" 입력

**방법 2: 푸시 버튼 사용**
- 상단의 **...** (점 3개) 메뉴 클릭
- **푸시** 선택

**인증:**
- Personal Access Token 입력 필요
- Username: `kiy0518`
- Password: `Personal Access Token 입력 (비밀번호가 아닌 토큰)`

## 📋 단계별 예시

### 예시 1: 문서 수정 후 업로드

1. `FIREFIGHTING_SYSTEM.md` 파일 수정
2. 소스 제어 패널 열기 (`Ctrl+Shift+G`)
3. `FIREFIGHTING_SYSTEM.md` 옆의 **+** 클릭 (스테이징)
4. 커밋 메시지 입력: "화재 진압 시스템: RTK GPS 섹션 추가"
5. **✓ 커밋** 클릭
6. **동기화** 버튼 클릭 (푸시)

### 예시 2: 여러 파일 수정 후 업로드

1. 여러 파일 수정
2. 소스 제어 패널 열기
3. **모두 스테이징** 버튼 클릭
4. 커밋 메시지 입력: "프로젝트 업데이트: Git 워크플로우 가이드 추가"
5. **✓ 커밋** 클릭
6. **동기화** 버튼 클릭

## 🔍 유용한 기능

### 변경사항 미리보기

- 파일 이름 클릭: 변경 내용 확인
- **+** 버튼: 변경사항을 스테이징 영역에 추가
- **-** 버튼: 스테이징 취소

### 커밋 히스토리 확인

- 소스 제어 패널에서 **...** 메뉴 클릭
- **커밋 기록 보기** 선택
- 또는 `Ctrl+Shift+P` → "Git: View History"

### 브랜치 관리

- 하단 상태바에서 현재 브랜치 확인 (`main`)
- 브랜치 이름 클릭하여 전환/생성 가능

## ⚙️ Git 설정 (선택사항)

### 자동 푸시 설정

1. `Ctrl+,` (설정 열기)
2. "git push" 검색
3. **Git: Push After Commit** 체크
   - 커밋 시 자동으로 푸시

### 자동 가져오기 설정

1. 설정에서 "git fetch" 검색
2. **Git: Auto Fetch** 활성화
   - 주기적으로 원격 저장소 변경사항 확인

## 🎨 상태 표시

### 파일 상태 아이콘

- **U**: 수정됨 (Modified)
- **A**: 추가됨 (Added)
- **D**: 삭제됨 (Deleted)
- **M**: 병합 충돌 (Merge conflict)

### 상태바

- 하단 상태바에 Git 상태 표시
- 커밋되지 않은 변경사항 개수 표시

## 💡 팁

### 1. 커밋 전 변경사항 확인

- 파일 이름 클릭하여 변경 내용 확인
- 잘못된 변경사항은 되돌릴 수 있음

### 2. 의미 있는 커밋 메시지

```
좋은 예:
- "RTK GPS: 정밀 호버링 기능 추가"
- "설치 스크립트: 네트워크 설정 자동화"
- "문서: QGroundControl 연결 가이드 추가"

나쁜 예:
- "수정"
- "업데이트"
- "asdf"
```

### 3. 부분 커밋

- 모든 파일을 한 번에 커밋하지 않고
- 관련된 변경사항끼리 묶어서 커밋
- 예: 문서 변경과 코드 변경을 분리

### 4. 정기적인 백업

- 작업 완료 시마다 커밋 및 푸시
- 하루 종료 시 백업 커밋

## 🔧 문제 해결

### "Authentication failed"

- Personal Access Token 확인
- 토큰이 만료되었으면 새로 생성
- GitHub → Settings → Developer settings → Personal access tokens

### "Updates were rejected"

- 원격 저장소에 새로운 변경사항이 있음
- **...** 메뉴 → **가져오기** 클릭
- 충돌 해결 후 다시 푸시

### "No changes to commit"

- 변경사항이 없음
- 모든 변경사항이 이미 커밋됨
